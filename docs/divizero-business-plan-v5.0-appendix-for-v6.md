## 0. 法人化のタイミング・軍資金・撤退リスク（ver.3.0 新設）

**v1.0・v2.0**: 本章の見出し・本文なし（法人化タイミングは未記載）。

**v3.0（新設）**:

### 法人成りに採用しない基準

- 世間一般で言われる **節税目的の売上800万円** という基準は**採用しない**。

### 法人成りを実行する条件（両方を完全に満たすタイミングのみ）

1. **信用の獲得とスケールアップの必要性**  
   事業を2倍・3倍へ拡大するプロセスにおいて、**社会的な信用（法人格）**があることでより大きな契約や展開が見込めるフェーズに到達したか。
2. **収益モデルの確実な立証**  
   現在のビジネスモデルにおける**収益性が完全に確認**でき、規模を拡大すれば**それに比例して収益も確実に増加する**という見込みが立ったか。

### 軍資金の確保と撤退リスクの管理

- **初期段階から法人化して身動きが取りにくくなるリスク**を避ける。
- まずは**機動力のある現在の形態**でテストマーケティング的に動き、**どこかの分野で確実に事業を確立**させる。
- **うまくいかなかった場合の撤退のしやすさ**を維持しながら、**運営の軍資金を積み上げる**ことを最優先とする。

### ver.5.0 追記：競合キッチンカーとの関係・「独占」の定義・学生の防衛線（論点メモ：ユーザー呼称「未確定変数4」）

**注**: **v1.0 §9 の未確定リスト番号4（保健所の壁）とは別枠**の論点である。

**v2.0 → v5.0（「独占」文言の補足と競合方針の明文化）**

- **v2.0 まで**: 「他のキッチンカーには同一場所を提供しない**独占**」と記載し、プラットフォーム経由の構造を説明。
- **v5.0（精緻化）**:
  - **既存のキッチンカー業者**には、相手が**個人か企業か**で対応の細部を変えつつ、基本は**共存**スタンスとする。**自分で場所の許可を取れるなら一緒にやってもよい**という整理。
  - **みかじめ料**のような**不当な要求は一切行わない**。
  - 事業計画における**「独占」**とは、**他主体を完全に排除すること**ではなく、**ディビゼロ側が優先的に選択する権利を得ること**を意味する、と定義し直す。
  - **学生の防衛線**: 学生の業務は**書類・手続きの進行**に特化し、**既存キッチンカー業者と現場で直接関わらせない**。
  - **クレーム・問い合わせ**は**専用の相談窓口（デスク）**に集約し、**学生に直接電話・連絡が行かない**仕組みを徹底する。

- 3ヶ月単位の契約とし、中途撤退時の返金は一切行わない。

**v1.0 → v2.0（モデル表現）**

- **v1.0**: 上記は **3分割の時間枠** を前提とした数式。
- **v2.0**: **実運用の枠数は 3 に固定しない**。収益の考え方（経費に対する転貸・手残り）は維持しつつ、**2 枠・4 枠等の設計**を §「時間枠・ゴールデンタイム」で補足（数式の「3枠合計150」は **3分割モデルとしての参照例** として残す）。

## 8. 決済と資金保留リスクの解決策（ver.4.0 新設）

**v1.0〜v3.0**: 本章見出しなし。§6 に送金経路の骨子のみ。

**v3.0 → v4.0**: 以下の方針で採用する（**特定商取引法・業種・契約形態・決済各社規約は都度法務・実務確認**すること）。

### 【1. 決済手段の優先順位とお客様を誘導する傾斜戦略】

お客様の**損失回避心理**を踏まえ、運営側にとって**安全で利益率が高い決済手段**へ自然に誘導する価格設計とする。

| 順位 | 手段 | 運営側の位置づけ | お客様への誘導・価格の考え方 |
|------|------|------------------|--------------------------------|
| **第1位** | **銀行振込（ネットバンク）** | 決済手数料ゼロ、**強制返金リスクを抑えやすい**ルート | 振込手数料はお客様負担とする。**いまネットバンクで振込いただけるなら、初期設定サポート費 1,000円を特別に無料**とする等で、**最優先でここへ誘導**する。 |
| **第2位** | **PayPay for Business** | 決済手数料 **1.98%**。スマホで完結し、**チャージバックリスクは相対的に低い**想定 | **標準価格**として提示する。特商法上の訪問販売規制との関係は、**BtoB の出張コンサルティング等の建付け**と**オンライン請求書＋決済リンク**の運用で整理する（**最終判断は法務**）。 |
| **第3位** | **Stripe（クレジットカード）** | 手数料 **おおよそ 3.6%**。**アカウント凍結・強制返金のリスクが相対的に高い** | カード希望者には **決済システム事務手数料として 500円上乗せ**し、上位2手段を**相対的にお得**に見せる。 |
| **第4位** | **コンビニ決済** | 現金主義の顧客向け | **代行収納手数料として 1,000円上乗せ**し、あえて割高にして**デジタル決済へ意識を向ける**。 |

### 【2. Stripe ハブ化の回避と自作決済ルーティングアプリの構築】

- **問題意識**: すべての決済計算を **Stripe ハブ**に寄せると、**銀行振込・PayPay まで Stripe の凍結等のリスクに巻き込まれる**懸念がある。
- **方針（採用）**: **Vercel** や **Supabase** 等を用い、**ディビゼロ専用の見積もり生成アプリ**を自作する。**プラン・割引の金額計算はアプリ内で完結**させる。
- **ルーティング**:
  - **銀行振込・PayPay**が選ばれた場合 → **Stripe API を通さず**、公式の**振込先・PayPay 支払い案内**へ誘導する。
  - **クレジットカード**が選ばれた場合のみ → **Stripe を呼び出す**。
- これにより、**カード決済ルートのリスクを他手段から隔離**する。

### 【3. ギルド兵による中抜きと悪用の完全封鎖】

学生が**個人口座・個人 PayPay**へ代金を誘導して**持ち逃げ**するリスクを、**システム**と**法務**の両面で抑える。

- **システム防衛**
  - 学生端末で**個人の QR コードを見せる**ことを**禁止**する。
  - 自作**見積もりアプリ**で確定金額を算出したうえで、**ディビゼロ公式 LINE 等**を経由し、**店主のスマホへ直接**決済リンク・**公式振込先**を自動送信するフローを徹底する。
- **心理的・法的防衛**
  - 店舗用パンフレットに、「担当者が**個人口座・個人 PayPay への送金を求めることはない**」旨と**通報窓口**を明記する。
  - 学生契約時に**身分証の提出を必須**とし、**不正受領は業務上横領・詐欺等として警察通報・退学リスク**を負いうることを**契約書に明記**して牽制する（**文面は弁護士等に確認**）。

### 【4. 究極の未回収対策：システムロックと完全自動化】

- 納品した Web に**課金の壁**を組み込み、**1週間の無料期間終了時**に**支払い画面のポップアップ**が出て**操作不能**になる**時限式**の仕組みを実装する（方針として採用）。
- **支払い**: **GMO あおぞらネット銀行**等の**バーチャル口座 API**や **Stripe** をバックエンドに組み込み、**着金を Webhook で検知**してデータベースのフラグを更新し、**自動でロック解除**する。
- **営業マンは金銭に触れない**。**システムが集金と解錠まで完結**させる設計とする。

## 1. 事業核心・ビジネスモデル

- **核心概念**: あらゆる高コスト・高障壁なビジネスを「分割（Division）」し、収益を生まないデッドタイムを「ゼロ（Zero）」にする汎用型OS。
